Utility Functions

ninja.escape (str)
String escaping. Escape a string such that it can be embedded into a Ninja file without further interpretation.

Parameters:

  • str string input string

Returns:

    string The escaped string
ninja.expand (str[, vars={}[, locals={}]])
Expand Expand a string containing vars as Ninja would. Doesn't handle the full Ninja variable syntax, but it's enough to make configure.lua's use of it work.

Parameters:

  • str string String to expand.
  • vars table "global" variables - searched if var is not found in locals (default {})
  • locals table "local" variables - first place searched for variables (default {})

Returns:

    string expanded string
ninja.as_list (input)
Converts input into a table of strings. If input is a blank string or nil, an empty table will be returned. If input is a single scalar value, it will be converted to a string and inserted into a table. Used internally

Parameters:

  • input input value

Returns:

    table input converted to a table of strings
ninja.escape_path (word)
Expanded version of ninja.escape. Used internally for stuff.

Parameters:

  • word string text to escape

Returns:

    string escaped version of word

    escape

Build File Writer

Writer.new (output[, width=78[, blanklines=1]])
Writer constructor

Parameters:

  • output string or file filepath or file handle for output .ninja file
  • width int maximum line width for wrapping (default 78)
  • blanklines int maximum consecutive blank lines (default 1)
Writer:newline ()
Outputs a newline. Blank lines are limited to the writer's blanklines field if said field is non-zero.

Returns:

    Writer returns self
Writer:comment (text[, has_path=false])
Outputs a comment. Comment text is automatically wrapped according to the writer's width.

Parameters:

  • text string Comment text
  • has_path bool Unused - exists to maintain consistency. (default false)

Returns:

    Writer returns self
Writer:variable (key, value[, indent=0])
Define a variable to its value Blank/nil values are ignored. (and filtered from collections)

Parameters:

  • key string variable name
  • value string, number, boolean or table variable value
  • indent int indent level (default 0)

Returns:

    Writer returns self
Writer:pool (name, depth)
Pool declaration Pools allow you to allocate one or more rules or edges a finite number of concurrent jobs which is more tightly restricted than the default parallelism.

Parameters:

  • name string pool name
  • depth int pool depth

Returns:

    Writer returns self
Writer:rule (name, command[, kwargs={}])
Rule declaration Possible kwargs keys include: description, deps, depfile, pool, rspfile, rspfile_content, generator, and restat.

Parameters:

  • name string rule name
  • command string rule command line
  • kwargs table additional optional rule configuration (default {})

Returns:

    Writer returns self
Writer:build (outputs, rule, inputs, implicit, order_only[, variables])
Build statement Build statements declare a relationship between input and output files.

Parameters:

  • outputs string or table output file(s)/target(s)
  • rule string declared rule to use
  • inputs string or table input file(s)/target(s)
  • implicit string or table implicit dependencies
  • order_only string or table force build to occur after target(s)
  • variables table any shadowed variables for this particular build (optional)

Returns:

    Writer returns self

Include statement Used to include another .ninja file into the current scope.

Returns:

    Writer returns self
